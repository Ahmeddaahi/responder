# Quick Fix: Verification Emails Not Sending

## 🎯 Most Common Issues (Check These First!)

### 1. ✅ Brevo Sender Email Not Verified (90% of issues)

**Check:**
1. Go to [Brevo Dashboard](https://app.brevo.com/) → **Settings** → **Senders**
2. Find your sender email
3. **Must show green checkmark "Verified"**

**Fix if not verified:**
- Check your email inbox
- Click the verification link from Brevo
- Wait 2-3 minutes
- Refresh the Brevo page

---

### 2. ✅ Wrong SMTP Password

**Check:**
- In Supabase, you need the **SMTP password**, not your Brevo account password
- SMTP password is a long random string generated by Brevo

**Fix:**
1. Go to Brevo Dashboard → **Settings** → **SMTP & API** → **SMTP**
2. Click **"Generate SMTP Password"** or **"Create SMTP Key"**
3. Copy the password (you won't see it again!)
4. Go to Supabase → **Settings** → **Auth** → **SMTP Settings**
5. Paste the new password
6. Save

---

### 3. ✅ Test Email in Supabase

**Do this test:**
1. Go to Supabase Dashboard → **Settings** → **Auth** → **SMTP Settings**
2. Click **"Send Test Email"**
3. Enter your email
4. Click **"Send"**

**Results:**
- ✅ **Test email received?** → SMTP works! Problem is elsewhere (see below)
- ❌ **Test email failed?** → SMTP configuration issue (fix Steps 1-2 above)

---

### 4. ✅ Check Spam Folder

- Brevo emails often go to spam initially
- Check spam/junk folder
- Mark as "Not Spam" if found

---

### 5. ✅ Email Confirmations Enabled?

**Check:**
1. Supabase Dashboard → **Settings** → **Auth**
2. Scroll to **"Email Auth"**
3. **"Enable email confirmations"** must be **ON**

---

### 6. ✅ Site URL & Redirect URLs

**Check:**
1. Supabase Dashboard → **Settings** → **Auth**
2. **Site URL** should be: `https://responder-eight.vercel.app`
3. **Redirect URLs** should include: `https://responder-eight.vercel.app/auth`
4. Add it if missing!

---

### 7. ✅ Brevo Daily Limit

**Check:**
1. Brevo Dashboard → **Statistics**
2. Free tier: **300 emails/day**
3. If exceeded, wait 24 hours or upgrade plan

---

## 🔍 Diagnostic Steps

### Step 1: Test SMTP Connection
```
Supabase → Settings → Auth → SMTP Settings → "Send Test Email"
```
- ✅ Works? → Go to Step 2
- ❌ Fails? → Fix SMTP configuration (Steps 1-2 above)

### Step 2: Check Supabase Logs
```
Supabase Dashboard → Logs → Auth Logs
```
Look for:
- Signup attempts
- Email sending errors
- SMTP connection errors

### Step 3: Check Brevo Dashboard
```
Brevo Dashboard → Statistics → Emails
```
Check:
- Are emails being sent?
- Delivery status
- Any bounces?

---

## 🛠️ Quick Fix Checklist

Run through this in order:

- [ ] Brevo sender email is **verified** (green checkmark)
- [ ] Using correct **SMTP password** (not account password)
- [ ] Supabase **test email works**
- [ ] Email confirmations **enabled** in Supabase
- [ ] Site URL and redirect URLs **configured correctly**
- [ ] Checked **spam folder**
- [ ] Not exceeding **Brevo daily limit** (300/day)
- [ ] Checked **Supabase Auth logs** for errors
- [ ] Checked **Brevo dashboard** for delivery status

---

## 🆘 Still Not Working?

### Check These Specific Settings:

1. **Supabase SMTP Settings:**
   ```
   Host: smtp-relay.brevo.com
   Port: 587 (or 465 for SSL)
   Username: your-brevo-account-email@example.com
   Password: [SMTP password from Brevo]
   ```

2. **Supabase Auth Settings:**
   - Enable email confirmations: **ON**
   - Site URL: `https://responder-eight.vercel.app`
   - Redirect URLs: Must include `https://responder-eight.vercel.app/auth`

3. **Brevo Settings:**
   - Sender email: **Verified** (green checkmark)
   - SMTP password: Generated and copied correctly

## 📞 Need More Help?

If you've checked everything above:

1. **Screenshot these:**
   - Supabase SMTP Settings (hide password)
   - Brevo Sender status
   - Supabase Auth Logs (last 10 entries)
   - Brevo Email Statistics

2. **Check error messages:**
   - Browser console (F12)
   - Supabase Auth Logs
   - Brevo Dashboard → Activity

3. **Common error messages:**
   - `535 Authentication failed` → Wrong SMTP password
   - `550 Sender not verified` → Sender email not verified
   - `Connection timeout` → Wrong host/port
   - `Rate limit exceeded` → Too many emails

## ✅ Success Indicators

You'll know it's working when:
- ✅ Supabase test email is received
- ✅ Signup creates account without 422 error
- ✅ Verification email arrives in inbox (or spam)
- ✅ User can click link and verify email
- ✅ User is redirected and logged in

## 🎯 Most Likely Fix

**90% of the time, it's one of these:**
1. **Sender email not verified in Brevo** → Verify it!
2. **Wrong SMTP password** → Regenerate and update!
3. **Email in spam folder** → Check spam!

Start with these three! 🚀
