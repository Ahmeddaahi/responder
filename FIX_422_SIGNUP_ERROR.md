# Fix 422 Signup Error - Quick Guide

## 🔴 Problem
You're getting a **422 error** when users try to sign up, and verification emails are not being sent.

## ✅ Solution
Configure Supabase to use Brevo SMTP for sending verification emails.

---

## 🚀 Quick Fix (5 Minutes)

### Step 1: Get Brevo SMTP Password

1. Go to [Brevo Dashboard](https://app.brevo.com/) → **Settings** → **SMTP & API** → **SMTP** tab
2. If you don't see an SMTP password, click **"Generate SMTP Password"** or **"Create SMTP Key"**
3. Copy the password (you won't see it again!)

### Step 2: Configure Supabase

1. Go to [Supabase Dashboard](https://supabase.com/dashboard) → Your Project
2. Navigate to **Settings** → **Auth** → **SMTP Settings**
3. Enable **"Enable Custom SMTP"**
4. Enter these values:

   ```
   Sender email: your-verified-email@yourdomain.com
   Sender name: Reply Ready Bot
   Host: smtp-relay.brevo.com
   Port: 587
   Username: your-brevo-account-email@example.com
   Password: [paste the SMTP password from Step 1]
   ```

5. Click **"Save"**

### Step 3: Test

1. Click **"Send Test Email"** in Supabase SMTP Settings
2. Enter your email
3. Check your inbox
4. If test email works, try signing up again!

---

## ⚠️ Important Notes

1. **Sender Email Must Be Verified:**
   - Go to Brevo Dashboard → Settings → Senders
   - Your sender email must show a green checkmark (verified)
   - If not verified, check your email and click the verification link

2. **SMTP Password ≠ Account Password:**
   - The SMTP password is different from your Brevo login password
   - It's a long random string generated by Brevo
   - You need to generate it specifically for SMTP

3. **Port Options:**
   - Use port **587** with TLS (recommended)
   - Or port **465** with SSL (if 587 doesn't work)

---

## 🐛 Still Not Working?

### Check These:

1. **Brevo Sender Verification:**
   - Brevo Dashboard → Settings → Senders
   - Must be verified (green checkmark)

2. **SMTP Password:**
   - Make sure you're using the SMTP password, not your account password
   - Generate a new one if needed

3. **Supabase Test Email:**
   - Try sending a test email from Supabase
   - Check the error message if it fails

4. **Brevo Limits:**
   - Free tier: 300 emails/day
   - Check if you've exceeded the limit

5. **Check Spam Folder:**
   - Verification emails might go to spam initially
